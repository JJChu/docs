# 成长记录

# 技术天花板

截止 2019 年，自我反思。感觉已经到了自我技术瓶颈期，技术开始停滞不前，每天的工作成了复制粘贴。对所有的知识都是浮于表面，理论不深刻，实践更为趋无。待在自己的舒适区始终无法突破，现沉思良久，针对自己的弱点，量身订造行为准则。

# 自我思考

  1. 缺乏技术驱动（不知道应该学习啥，学到什么程度，能用到哪里）
  2. 缺乏有效的方法论（不知道怎么去学，怎么实践，学过即忘，用时一知半解）
  3. 格局太小（满足于现状和技能，思维短浅）

# 行为准则

缺乏技术驱动力

找不到技术驱动力，往往是因为我们总是呆在自己的舒适区。我们知道：能力越大，责任也越大。但我更要知道：责任越大，能力也要越大。所以责任可以驱动我们去获取更大的能力。所以，在工作中，除了业务代码，我们要从更多的层面去思考，比如 vue 的实现原理，swiper 的实现，如果抽象代码等等。我们要从更多层面去改善自己的编程能力，而不只是技能。

缺乏有效的方法论

关于如何学，一直是很多人都会有的一个难题。我们总是学完忘，忘了学，周而复始。一个有效的方法论能够让我们事半功倍。现总结如下：

  • 勇于探索，提取主干；（对于未知领域我们一定不要犯怵，所有的东西最后做的事情都很简单。比如增量编译，说白了就是依据变更文件反推出所有依赖文件并重新编译）
  • 依据主干，反推路线；（知道了主干，我们就很自然地知道了我们需要学习的内容，语言只是工具，思维才是能力）
  • 遇到问题，自己先做；（自己先做的会给我们留下很多思考的空间，我们会积攒更多的问题去解答）
  • 化面为点，简化问题；（所有问题我们都应该找到其核心内容，尤其是算法，抽丝剥茧找到主体，举例算法）
  • 总结要点，强化思考；（横向对比，纵向对比，对比记忆，总结共性，强化思考）
  • 自我说服，多思多练；（能够对别人讲明白才是真正内化知识的表现，同时一定要多练多练多练）

古人云：每天坚持 2 小时的变成和 2 小时的思考，10000 个小时可以让你成为一个领域专家

所以每天要抽出 1 到两小时去根据自己的规划头脑风暴，依据自己的方法论去总结。

格局太小

格局是一个程序员从到工程师必备的思维，一个人的格局决定了他的成长。一个程序员需要拥有什么样的格局，需要我们自我探索和高手们的持续交流中去体会总结。

# 良好心态

# 1. 走出舒适区

  • 舒适区 - 使我们能力范围内的区域，在这个区域，我们很舒服没有特别大的心理压力。在这个区域哪怕你每天很忙，你的成长也很缓慢；
  • 学习区 - 能力范围之外，但经过努力或他人帮助，我们也能够搞定的区域。在学习区，我们感觉到压力、焦虑，但在这个区域，能充分的锻炼自我，挑战自我，提升能力。
  • 恐慌区 - 顾名思义，这个区域完全超出了个人能力，即使努力也搞不定，待在这个区域会让人惊慌失措，悲观绝望

所谓刻意练习，就是在学习区内进行的大量练习，不断将学习区变成舒适区，将恐慌区变成学习区，不断将上面那张图向外拓展。

# 2. 以平常心看待比自己优秀的人

# 3. 不在深夜做决定

# 5. 面对任何选择，牢记“我需要”，“我喜欢”，“我适合”这三大筛选标准

# 优秀思维

# 1. 行动思维

对于未知领域或者新知识，我们总是想着先学后做，害怕知识不完整，其实实践比从零开始更有效率，实践可以帮我们更快地找到弱项，而且动手得到的东西比我们死记硬背的更透彻。不想拖延的事情，立即做5分钟

# 2. 透过现象看本质的思维

我们总是羡慕那些聊天时能侃侃而谈，或者应激反应很厉害的人。其实他们都有一个特性，透过现象看本质。有这个习惯的人他们对所有的事情都能很快检索到本质信息，比如大家都喜欢刷抖音，但没人思考背后成瘾的原因；大家都喜欢聊八卦，但没人探索背后的资本运作和流量引导。

你看到了本质，就不会轻信任何一种言论，不会那么容易随波逐流地成为“乌合之众”

从现在开始就注意两件事：

  1. 看到一样事物后突然迸发了灵感，那么停下手上非紧急的事，顺着你的灵感去思考，并记录和执行；
  2. 对于遇到的人和事，多追问自己几个为什么，尽量去找问题的核心和本质

# 分而治之（聊胜于无）（Divide and Conquer）

在遇到一个比较困难的问题时，首先想办法把问题拆解开，用最快的时间找出其中最容易的部分先实现出来。这样，虽然离真正解决问题还很远，但是也已经比“知难而退”或者是“拖延”都要好的多了。抽象并实现一些简单问题之后，或许你还会发现困难的问题也变得更好条理了

# 结构化思维

人类大脑在处理信息的时候，有两个规律：第一，不能一次太多，太多信息会让我们的大脑觉得负荷过大；第二，喜欢有规律的信息。

如果我们在解决问题、面临选择以及与人沟通的时候，能够找到一个结构，将所有的碎片信息放进去，就能够大大减轻大脑的负担，更容易地解决问题。而这，就是所谓的“结构化思维”。

两种方法：

1. 自上而下找结构：思考一个框架，然后将信息或解决方案放入框架

比如，我们要对本周所有工作做一个汇总，这个时候我们可以使用四象限工作法，按照紧急性和重要性把任务填充进去。然后我们就知道我们应该怎么组了；

再比如，对某个东西的宏观分析，宏观环境分析比较常见的结构是 PEST（Political政治层面, Economic经济层面, Social社会层面, Technological技术层面）你可以用这个结构把你的观点都放进去。

所以，针对一些常见问题，你可以很容易地找到一个结构，将思考结构化。可是当你脑海中缺乏现成的结构时，会变得非常困难。

常用的一些结构类型：

  • 二维矩阵类：任务分析矩阵（紧急性、重要性）、波士顿矩阵（市占率、增长率）、安索夫矩阵（新/老产品、新/老市场）
  • 流程类：产品价值链、客户生命周期
  • 三要素类：3C战略三角（Corporation公司, Customer顾客, Competition竞争对手）、爱情三元论
  • 利益相关方类：波特五力模型

2. 自下而上归纳总结成结构：当你不清楚可以用什么框架的时候，四种方式把信息结构化

  • 第一步：头脑风暴：拿一张白纸，将自己所有的想法碎片记录下来（溯源大法），列完之后，对类似的连线合并；
  • 第二步：连线分组，根据连线的情况，将想法进行分组
  • 第三步：结构提炼，看一下这些分组，是否存在某种规律，按照这样的规律，是否需要补充或调整组。这时候需要用到 MECE（Mutually Exclusive, Collectively Exhaustive，即相互独立、完全穷尽）原则，来决定分组调整。
  • 第四步：观点补充
