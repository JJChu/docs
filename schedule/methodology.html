# 如何阅读源码

大部分程序员都会“写”代码，或者至少会抄代码和改代码。但是，会读代码的并不在多数，会读代码又真正读懂一些大项目的源码的，少之又少。这种怪状，真要追究起来，怪不得程序员这个群体本身 —— 它是两个原因造成的：

  • 我们所有的教育和培训都在强调怎么写代码，并没有教大家如何读代码
  • 大多数工作场景都是一个萝卜一个坑，我们只需要了解一个系统的局部便能开展工作，读不相干的代码，似乎没用

那我们为什么学会读源码，确切的说，是优秀的源码。

# 为什么要阅读优秀代码

# 1. 积累素材

程序员这个行业的特殊性，致使我们“学习永不止步”。在 IT 领域，技术的更新迭代速度远远超过你掉头发的速度，所以，没有哪个人能够抱着一个饭碗吃到死，我们需要不断地补充我们的知识体系和技能。还有一个特点，我们接触的信息大多零碎且杂，我们逛掘金，看博客，但这些东西都是“对点不对面”，不成系统，看过即忘，难以形成可以体系化的知识。

写代码和写作有异曲同工之妙，在以前我们为了提升自己的写作能力，阅读大量优秀文学作品，甚至背诵摘抄引用，我们学习里面的修辞语法，优秀词汇，表达方式，终于在遇到一位妙龄女子，想到的是肌肤胜雪，明眸善睐，桃腮带笑，齿如含贝... 而不是简单的卧槽，真美。写代码也是这样，对比一些初级和高级程序员的代码，我们可以发现明显的差别，高级程序员在变量命名，代码组织方式，代码格式，实现方式上让人耳目一新，有阅读小说的快感，而有些人的代码就是一团糟。

优秀的库都是为解决某类问题而诞生的，我们可以从中积累解决问题的方案和思想，从而在使用的时候更加得心应手，甚至根据自己的业务需求进行改造；而且因为技术知识通常庞大且杂，尤其在前端，所以如何把零碎的知识体系化，或者说把零碎的信息整理成系统化的知识，这些在成熟的库中更能体现出来。比如，在 jquery 中，就包含了各种设计模式、网络等等知识。

# 2. 学习最佳实践

我们经常会有这些的疑惑，我封装的组件应该提供哪些接口？别人为什么一眼就能看出来问题？别人的实现方式好巧妙？诸如此类，首先恭喜你，你开始从“面向自己”编程向“面向大众”编程转换。程序员不仅要会写代码，还要写好代码，写别人看得懂的代码，这就是最佳实践。

优秀的类库中都包含了大量的最佳实践，提供哪些接口和配置，使用什么设计模式，怎么去抽象精简代码等，这些是我们要着重去学习和体会的。这些就好比功夫里面的心法，招式是死的，但心法是活的，心法指导你怎么去组织招式。

# 总结

累积素材是基础，被启发出来的思路将这些素材串成线，这就形成了知识（knowledge）。书读得越多，越勤于思考的人，知识也就越丰富。而知识的融会贯通，最终形成读书的第三大功用：通过了解，吸收别人的思想，去芜存菁，最终形成自己的思想，或者说智慧（wisdom）。

information -> knowledge -> wisdom 是个长期的累积，并非一朝一夕之功。
